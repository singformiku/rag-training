The CAP theorem states that a distributed data store can provide at most two of three guarantees simultaneously: **Consistency** (all nodes see the same data at the same time), **Availability** (every request receives a response, successful or not), and **Partition tolerance** (the system continues operating despite network partitions). Consequently, when a network split occurs, designers must choose between keeping the data consistent (sacrificing availability) or keeping the system responsive (sacrificing consistency).
