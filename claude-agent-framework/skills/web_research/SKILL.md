# Web Research Skill

## 研究流程

### 1. 拆解問題
把大問題拆成 3-5 個具體的子問題。例如：
- 大問題：「電動車市場現況如何？」
- 子問題：
  - 全球銷量趨勢？
  - 主要廠商市占？
  - 法規動向？
  - 技術瓶頸？

### 2. 多輪搜尋
- 每個子問題用 2-3 個不同的關鍵字搜尋
- 避免在單一來源上停留太久
- 優先考慮：官方報告、學術論文、主流媒體

### 3. 交叉驗證
當你看到一個數字或結論：
- ❌ 不要直接採信
- ✅ 找第二個獨立來源佐證
- ✅ 注意資料的時間範圍

### 4. 整理輸出
```
## 核心發現
- 重點 1 (來源：xxx)
- 重點 2 (來源：yyy)

## 詳細分析
(分段說明)

## 資料來源
1. [Title](URL) - 發布日期
2. [Title](URL) - 發布日期

## 尚未釐清的問題
- 還有哪些問題需要進一步查證
```

## 注意事項
- 標註資料的「時效」(2024 或 2025 差很多)
- 區分「事實」和「觀點」
- 對商業來源保持警覺 (廠商自己發的報告會美化數據)
