# Code Review Skill

當你被要求審查程式碼時，請依循以下流程：

## 1. 先理解脈絡
- 用 `read_file` 讀取目標檔案
- 若有相關檔案 (例如 test、同模組其他檔案)，也一併讀取
- 用 `list_directory` 了解專案結構

## 2. 審查維度 (由重要到次要)
1. **正確性**：邏輯錯誤、邊界情況、null check
2. **安全性**：SQL injection、path traversal、secrets 外洩
3. **可讀性**：命名、註解、函式長度
4. **效能**：不必要的迴圈、重複計算
5. **風格**：PEP 8 (Python)、專案既定風格

## 3. 回報格式
請用以下結構回報發現：

```
## 🔴 嚴重問題 (必須修正)
- [檔案:行號] 問題描述 + 建議修正

## 🟡 改進建議 (值得修正)
- [檔案:行號] 問題描述 + 建議修正

## 🟢 優點
- 值得肯定的寫法

## 💡 重構建議
- 更大範圍的架構調整建議
```

## 原則
- 針對「程式碼」而非「人」提出回饋
- 每個問題都附上具體的修改範例
- 解釋「為什麼」，不只說「要改」
